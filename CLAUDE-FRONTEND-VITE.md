# Vite Frontend Conventions

Framework-specific rules for Vite React single-page apps. Read together with `~/.claude/CLAUDE-FRONTEND.md` (the shared core); everything not covered here follows the core.

---

## Framework

- **Vite** with **React 19** as a client-rendered SPA; no SSR
- **TanStack Router** with file-based routing (`src/routes/`)
- `routeTree.gen.ts` is generated by the TanStack Router Vite plugin; commit it, never hand-edit it

---

## Directory Structure

```
src/
├── main.tsx                  # Entry: createRoot + RouterProvider, nothing else
├── routeTree.gen.ts          # Generated; never hand-edited
├── routes/                   # TanStack Router file-based routes
│   ├── __root.tsx            # Root route: providers, layout shell
│   ├── index.tsx             # Landing / index page
│   ├── login.tsx
│   └── _protected/           # Pathless layout route for authed pages
│       ├── route.tsx         # Auth guard + shared authed layout
│       └── dashboard.tsx
├── components/               # Shared UI components (see core)
├── features/                 # Feature slices (see core)
├── api/                      # Own-backend fetch wrappers (see core)
├── clients/                  # Third-party SDK wrappers (see core)
├── services/                 # Domain logic (see core)
├── state/                    # Stores, hooks, context providers (see core)
├── config/                   # queryClient.ts, router config, env parsing
├── constants/
├── data/
├── styles/                   # globals.scss, design tokens
└── types/
```

### Rules

- Everything outside `routes/`, `main.tsx`, and `routeTree.gen.ts` follows the shared directory vocabulary in the core file
- Directories appear only when occupied (R-309); the vocabulary is fixed, not mandatory on day one
- Route URL segments are kebab-case (`routes/coming-soon.tsx`) per the R-312 exception; every other directory is camelCase
- Pathless layout routes use the `_prefix` form (`_protected/`); the auth guard lives in that group's `route.tsx`
- Route files stay thin: export the route definition and a small component that delegates to `features/` or `components/`
- `__root.tsx` holds app-wide providers (QueryClientProvider, context providers) and the layout shell
- Global styles import once in `main.tsx` (`import './styles/globals.scss'`)

---

## Entry Chain

- `index.html` at the project root is the single HTML entry; it loads `/src/main.tsx`
- `src/main.tsx` does exactly: create the router, mount `<RouterProvider>` with `createRoot`; no other bootstrap logic
- Client-only rendering: no `'use client'` directives anywhere; every module is client-side

---

## Environment Variables

- `VITE_*` prefix, read via `import.meta.env`
- Parse and validate env values in `config/` (a `config/env.ts` module); components never touch `import.meta.env` directly
- API base URL comes from `VITE_API_URL`

---

## Path Alias

- `@/` maps to `src/`, configured in both `vite.config.ts` (`resolve.alias`) and `tsconfig.json` (`paths`)
- Import ordering group 1 (see core) is React plus `@tanstack/react-router` imports

## File Naming (framework-specific rows)

| What | Convention | Example |
|------|-----------|---------|
| Routes | kebab-case route path | `routes/coming-soon.tsx` |
| Root route | `__root.tsx` | `routes/__root.tsx` |
| Pathless layout | `_name/route.tsx` | `routes/_protected/route.tsx` |
| Entry | `main.tsx` | `src/main.tsx` |
| Global styles | `globals.scss` | `styles/globals.scss` |
